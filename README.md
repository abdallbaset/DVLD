  ### 📖 نظرة عامة (Overview)
  هذا المشروع يتجاوز فكرة واجهات المستخدم التقليدية، حيث يركز على بناء **Presentation Layer** ذكية تعمل كخط دفاع أول لضمان صحة البيانات. يتمحور المشروع حول **منع الأخطاء البشرية** قبل وصولها إلى قاعدة البيانات، مع نظام أمان متكامل لإدارة المستخدمين وصلاحياتهم.

  > **ملاحظة للمستعرض:** الواجهات الحالية (UI) صُممت لتكون وظيفية لاختبار المنطق البرمجي، وسيتم تطوير واجهة احترافية (Modern UI) في إصدارات لاحقة.

  ---

  ### 🔐 نظام الوصول والأمان (Login & Access)
  يعتبر نظام تسجيل الدخول هو البوابة الأمنية للنظام, ويتميز بـ:
  * 🔑 **التحقق من الهوية:** مطابقة اسم المستخدم وكلمة المرور مع قاعدة البيانات المشفرة.
  * 🛡️ **فحص حالة الحساب (Is Active):** نظام حماية يمنع دخول أي مستخدم تم تعطيل حسابه برمجياً حتى لو كانت كلمة المرور صحيحة.
  * 💾 **خاصية "تذكرني" (Remember Me):** إمكانية حفظ بيانات الدخول مشفرة في ملفات النظام المحلية لتسهيل الدخول المستقبلي.
  * 👤 **جلسة المستخدم (Global User Session):** بمجرد الدخول، يتم تخزين بيانات المستخدم في "كائن عالمي" (Global Object) لتتبع العمليات التي يقوم بها (مثل: من الذي قام بإصدار هذه الرخصة؟).

  ### 🛠️ الوظائف الأساسية (Core Functionality)
  يوفر النظام إدارة كاملة من خلال العمليات التالية:

  #### 👥 إدارة الأشخاص (People Management)
  * **إضافة/تعديل:** إدخال البيانات الشخصية مع دعم رفع الصور وحفظها تلقائياً.
  * **عرض البطاقة:** استعراض التفاصيل في `Person Card` منظمة.
  * **الحذف الذكي:** حذف السجل مع ميزة "التنظيف التلقائي" لصور الأشخاص من القرص الصلب.

  #### 👤 إدارة المستخدمين (User Management)
  * **ربط البيانات:** لا يمكن إنشاء مستخدم إلا إذا كان "شخصاً" مسجلاً مسبقاً في النظام لضمان وحدة البيانات.
  * **التحكم في الحساب:** إمكانية تفعيل أو تعطيل حساب المستخدم (Active/Inactive) دون الحاجة لحذفه.
  * **تصفية ذكية:** نظام فلترة متقدم للبحث عن المستخدمين بواسطة (UserID, UserName, FullName, Status).

  #### ⚙️ إعدادات الحساب (Account Settings)
  * **تغيير كلمة المرور:** واجهة مخصصة تفرض التحقق من كلمة المرور الحالية قبل السماح بتعيين كلمة مرور جديدة.
  * **معلومات المستخدم:** عرض تفصيلي لبيانات المستخدم الحالي وصلاحياته.
  * **تسجيل الخروج (Logout):** إنهاء جلسة المستخدم الحالية بأمان وتصفير بيانات الـ `Global User`.
    * العودة التلقائية لشاشة تسجيل الدخول لإتاحة المجال لمستخدم آخر.

  #### 📋 إدارة طلبات الرخص (Application Management)
  * **طلبات القيادة المحلية (Local Driving License Applications):** إدارة كاملة لدورة حياة طلب الرخصة المحلية بدءاً من تقديم الطلب، مروراً بالفحوصات وحتى إصدار الرخصة.
  * **أنواع الطلبات والرسوم:** تسيير ديناميكي لأنواع الطلبات (تجديد، استبدال، أول مرة) مع جلب قيم الرسوم الخاصة بكل خدمة مباشرة من قاعدة البيانات.
  * **التحكم في حالات الطلب (Application Status):** تتبع حالة الطلب وتحديثها تلقائياً (جديد، مكتمل، ملغي) مع تسجيل تاريخ آخر تحديث للحالة لوثوقية البيانات.

  #### 📝 نظام الاختبارات الذكي (Test Appointment System)
  * **الفحوصات الثلاثية المترابطة:** تنظيم واختبار المتقدمين عبر 3 مراحل متتالية لا يمكن تخطي إحداها إلا بالنجاح في السابقة:
    1. **الفحص الطبي (Vision Test):** فحص النظر الأساسي وتوثيق موعده ونتيجته.
    2. **الفحص النظري (Written Test):** اختبار المعلومات والقوانين بعد اجتياز فحص النظر.
    3. **الفحص العملي (Street Test):** الاختبار الميداني النهائي لقيادة المركبة.
  * **إدارة المواعيد (Test Appointments):** حجز وإعادة حجز المواعيد للاختبارات مع قفل الحجز بمجرد تقييم الاختبار (Pass/Fail) لمنع التلاعب بالنتائج.
  * **رسوم إعادة الاختبار (Retake Test Fees):** احتساب رسوم إضافية تلقائياً في حال رسوب المتقدم ورغبته في حجز موعد اختبار جديد.

  #### 🪪 إدارة رخص القيادة (Licenses Management)
  * **إصدار الرخصة لأول مرة:** توليد رخصة قيادة جديدة وربطها بالطلب المكتمل وتحديد تاريخ انتهاء الصلاحية بناءً على نوع فئة الرخصة (License Class).
  * **تجديد الرخص المحلية (Renew License):** معالجة طلبات تجديد الرخص المنتهية أو المشرفة على الانتهاء، مع فحص الحالات الأمنية والصحية، واحتساب الرسوم الكلية (رسوم الطلب + رسوم الفئة).
  * **استبدال الرخص التالفة والمفقودة (Replacement for Damaged/Lost):** واجهة ذكية تتيح استبدال الرخصة مع تغيير أسباب الإصدار (Issue Reason) ورسوم الخدمة ديناميكياً بناءً على حالة الرخصة (تالفة أم مفقودة).
  * **الرخص الدولية (International Licenses):** إمكانية إصدار رخصة دولية معتمدة بناءً على رخصة محلية نشطة وصالحة، وتوليد رقم رخصة دولي خاص بها يربط العميل دولياً.
  * **سجل تاريخ الرخص (Driver License History):** شاشة مخصصة تعرض التاريخ الكامل والمفصل لجميع الرخص الصادرة للسائق (المحلية والدولية) وحالاتها الحالية للتتبع السريع.

  #### 🔒 نظام حجز الرخص وفك الحجز (Detain & Release Licenses)
  * **حجز الرخص (Detain License):** إمكانية سحب وحجز الرخص المخالفة مع تحديد قيمة الغرامة المالية (Fine Fees) وتوثيق هوية المستخدم الذي قام بالحجز وتاريخه.
  * **فك حجز الرخص (Release License):** منطق برمجي صارم يمنع السائق من استعادة رخصته المحجوزة إلا بعد دفع الغرامات المقررة عبر إنشاء "طلب فك حجز" مالي رسمي وتحديث حالة الحجز ليكون (Released).
  * **التحقق من حالة الحجز الفوري:** فحص تلقائي عند استعلام أي رخصة للتأكد مما إذا كانت محجوزة أم لا لمنع أي إجراء موازٍ عليها قبل فك الحجز.

  ### 🛡️ ضوابط التحقق في طبقة العرض (Presentation Layer Validation)

  #### 👤 منطق إدارة المدخلات
  * ✅ **التحقق الشامل قبل الحفظ:** فحص كافة الحقول المطلوبة وتنبيه المستخدم برسائل دقيقة (Error Provider).
  * ✅ **منع التكرار الفوري:** التحقق من عدم تكرار "الرقم الوطني" للأشخاص أو "اسم المستخدم" للمستخدمين فور الإدخال.
  * ✅ **قيود لوحة المفاتيح (Input Masks):** حقول الأرقام (هاتف، ID، قيمة الغرامة والرسوم) مبرمجة لرفض أي مفتاح غير رقمي برمجياً من خلال حدث الـ `KeyPress`.
  * ✅ **منطق السن القانوني (+18):** ضبط أداة اختيار التاريخ برمجياً لمنع تسجيل أي شخص يقل عمره عن السن القانوني للترخيص.
  * ✅ **تطابق كلمة المرور:** فحص فوري لتطابق كلمة المرور مع حقل التأكيد (Confirm Password).
  * ✅ **التحقق من سلامة الحالات الأمنية:** منع تجديد الرخص غير المنتهية، أو حجز الرخص المحجوزة مسبقاً، أو التعامل مع الرخص غير النشطة (Inactive) بفضل شروط فحص صارمة على مستوى الواجهات.

  ### 🏗 هندسة النظام (Architecture)
  تم بناء المشروع باستخدام هيكلية **N-Tier Architecture** لضمان فصل المهام، مع تطبيق معايير الأداء العالي وحماية البيانات:
  * **`DVLD_UI`**: خط الدفاع الأول (Validation, Logic Control) والتحقق الشامل عبر `ValidateChildren`.
  * **`DVLD_Business`**: طبقة منطق العمل (Brain) وإدارة القواعد المركزية، مبرمجة بالكامل بأسلوب البرمجة كائنية التوجه (OOP) والخصائص المختصرة `=>`.
  * **`DVLD_DataAccess`**: تنفيذ استعلامات SQL والتعامل مع قاعدة البيانات، مصممة بطريقة **(Single Database Round-Trip)** لدمج العمليات المترابطة (مثل التجديد والحجز) وتحقيق أقصى سرعة وأمان للبيانات.
  * **`DVLD_Model`**: كائنات تمثيل البيانات (DTOs) للتنقل السلس والآمن للبيانات بين الطبقات.
